# v1.0.3 - Add APU music channel timing

Release date: 2026-07-02

Tag: `v1.0.3`

Compare: `v1.0.2...v1.0.3`

## Summary

This release targets the remaining metallic edge in game music. Sound effects
were already much better after `v1.0.2`, so this pass avoids heavier global
filtering and instead adds the first NES-style timing controls that music relies
on: envelopes, length counters, and triangle linear counter gating.

The APU is still approximate, but held melody notes now have more natural
volume motion and channel gating instead of staying at fixed register volume.

## Changelog Since v1.0.2

### Audio

- Added approximate quarter-frame and half-frame APU timing.
- Added pulse-channel envelope decay support for music notes that use
  non-constant volume.
- Added pulse-channel length counters so notes can expire instead of droning
  indefinitely.
- Added noise-channel envelope decay and length-counter support.
- Added triangle-channel length-counter gating.
- Added triangle linear-counter reload and quarter-frame clocking.
- Kept the `v1.0.2` PolyBLEP pulse smoothing, smoother triangle waveform, noise
  smoothing, soft clipping, and pinned SDL mixer format.

### Code Documentation

- Documented envelope, length-counter, linear-counter, and frame-counter helper
  methods in the APU code.
- Kept comments focused on why the audio timing exists: it shapes music at the
  channel level instead of hiding artifacts with more output filtering.

### Packaging And Releases

- Bumped package version to `1.0.3`.
- Updated README release examples, artifact names, and release tag commands for
  `v1.0.3`.
- Added this permanent release note and linked it from the release index.

### Tests

- Added tests for pulse envelope decay and length-counter behavior.
- Added tests for triangle linear-counter gating.
- Updated version metadata and release documentation coverage for `v1.0.3`.

## Known Limitations

- CPU, PPU, and APU behavior are still not cycle-accurate.
- Pulse sweep units, exact APU frame-counter modes, DMC, IRQ behavior, and
  complete hardware sequencing are not implemented yet.
- The new timing should make music less static and metallic, but this is still
  not a full NES audio core.
- Real ROM smoke tests remain manual because ROMs are not committed to the
  repository.
