# v0.0.6 - Improve emulator frame performance

Release date: 2026-07-01

Tag: `v0.0.6`

Tag commit: `7aebc02`

Compare: `v0.0.5...v0.0.6`

## Summary

This release reduces per-frame overhead so ROMs can run closer to intended
speed. It focuses on normal-speed playability rather than artificially speeding
up gameplay.

## Added

- `APU.generate_samples_into()` for appending samples into an existing frame
  buffer.
- Cached tile-row slices for faster full background redraws.
- Regression tests for framebuffer byte reuse and repeated PPU writes that
  should not dirty the background cache.

## Changed

- `NES.run_frame()` reuses local CPU/APU/PPU references in the hot loop.
- Audio generation avoids allocating tiny byte strings per CPU instruction.
- The desktop renderer avoids copying packed framebuffer bytes before handing
  them to pygame.
- Sprite rendering skips offscreen and blank rows earlier.
- Sprite pixels are written inline instead of through an extra helper call.
- Background cache invalidation ignores repeated scroll, nametable, and palette
  writes that do not change rendered state.

## Fixed

- Static or lightly changing scenes avoid unnecessary redraw and audio-buffer
  churn.
- Sprite drawing and background redraws cost less CPU time.

## Validation

- `python -m pytest`
- `python -m ruff check .`
- `python -m build`
- `python -m nes_py --help`
- standalone executable build
- standalone executable `--help` and `--version` smoke checks

## Known Limitations

- Python rendering is still slower than a native or vectorized renderer.
- Games that heavily mutate scroll or VRAM can still force expensive redraws.
- APU accuracy remains approximate.
