# v1.0.0 - Playability and standalone release milestone

Release date: 2026-07-02

Tag: `v1.0.0`

Compare: `v0.0.1...v1.0.0`

## Summary

This release is the first playability-focused milestone for `pynes`. Since
`v0.0.1`, the emulator gained frame-paced execution, audio playback, sprite
rendering, scrolled backgrounds, mapper support improvements, standalone
executables, release documentation, and runtime catch-up behavior for overloaded
frames.

The emulator is still not cycle-accurate, but it can now load supported iNES
ROMs through a simple Windows or Linux executable, display backgrounds and
sprites, play approximate APU audio, and remain more responsive during busy
screen transitions.

## Changelog Since v0.0.1

### Runtime And Playability

- Added frame-paced execution with `NES.run_frame()`.
- Changed the desktop runner to execute one emulated video frame per draw by
  default.
- Added bounded adaptive frame skipping for overloaded desktop frames. The
  emulator can skip rendering up to two catch-up frames while CPU, PPU timing,
  input, and APU sample generation continue advancing.
- Added a render-skipping path through `NES.run_frame(render=False)` and
  `PPU.step(..., render=False)`.
- Swapped default controls so `Z` maps to NES `A` and `X` maps to NES `B`.
- Kept `--instructions-per-frame` for deterministic debugging and manual
  validation.

### Video

- Added OAM sprite rendering for 8x8 sprites.
- Added sprite palette lookup, transparency, horizontal and vertical flips, and
  basic priority handling.
- Added `$4014` OAM DMA support.
- Added scroll-aware background rendering using `PPUSCROLL`.
- Added PPUCTRL base-nametable selection for backgrounds.
- Added cartridge mirroring awareness for nametable reads.
- Added cached scrolled backgrounds and packed RGB background bytes.
- Avoided dirtying the background cache for repeated scroll, nametable, and
  palette writes that do not change rendered state.
- Added cached tile-row slices for faster full background redraws.
- Added an eight-sprites-per-scanline render cap, matching NES hardware behavior
  and reducing overdraw in crowded scenes.
- Fixed the Windows scaled-rendering crash caused by passing an incompatible
  destination surface to `pygame.transform.scale()`.

### Audio

- Added initial pulse-channel PCM generation.
- Added triangle and noise channel approximations.
- Added NES-style pulse/TND mixing formulas.
- Reduced per-instruction audio allocations by appending samples into a single
  frame buffer.
- Added a light output low-pass filter after DC blocking to reduce harsh,
  metallic edges.
- Reduced final output gain to avoid clipping and preserve melody shape.
- Reset audio filter state during APU reset.

### Cartridge And Compatibility

- Added Mapper 1 / MMC1 cartridge support.
- Improved unsupported cartridge errors so bad or unsupported ROMs fail with a
  user-facing message.
- Continued support for Mapper 0 and Mapper 2.

### Packaging And Releases

- Added standalone Windows and Linux executables through PyInstaller.
- Added manual release workflow dispatch.
- Added continuous build metadata, generated changelogs, and generated release
  notes for workflow artifacts.
- Added permanent release changelog docs for released versions.
- Bumped package version to `1.0.0`.

### Performance

- Added lightweight profiling tooling and performance notes.
- Optimized CPU opcode dispatch through a 256-entry dispatch table.
- Made package and CLI imports lazier for version/help paths.
- Reused packed framebuffer bytes when handing frames to pygame.
- Cached rendered backgrounds and tile rows to reduce redraw cost.
- Skipped unnecessary sprite work for offscreen rows, blank rows, and scanlines
  beyond the NES eight-sprite hardware limit.

### Tests And Validation

- Added tests for frame execution, skipped-render frames, audio output, OAM DMA,
  sprite rendering, sprite scanline limits, scrolled backgrounds, background
  cache reuse, scaled rendering, controls, version metadata, release docs, and
  standalone build behavior.
- Continued CI validation across Windows, Linux, and macOS.

## Known Limitations

- CPU, PPU, and APU behavior are still not cycle-accurate.
- APU envelopes, sweeps, length counters, and DMC are not complete.
- PPU scrolling, screen transitions, sprite priority, and mapper behavior remain
  simplified.
- Mapper support is still limited.
- Some games may still run slowly during scenes that mutate large amounts of PPU
  state every frame.
- Real ROM smoke tests remain manual because ROMs are not committed to the
  repository.
