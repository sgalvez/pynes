# v0.0.1 - Initial project scaffold

## Summary

This is the first release of the Python NES emulator project.

This release establishes the repository foundation, packaging metadata, CLI
entrypoint, tests, linting, CI, and release workflow.

## Included

- Python package scaffold under `src/nes_py`
- CLI entrypoint with help and version output
- Basic logging configuration
- Pytest test setup
- Ruff linting setup
- GitHub Actions CI
- Cross-platform validation for Windows, Linux, and macOS
- Python package build artifacts:
  - source distribution
  - wheel distribution
- Standalone executable artifacts:
  - Linux CLI executable
  - Windows CLI executable
- Early emulator modules for cartridge loading, CPU, bus, PPU, input, and
  debugging

## Not yet implemented

- Full NES game compatibility
- Complete mapper coverage
- Cycle-accurate CPU, PPU, and bus timing
- Sprite rendering
- APU/audio
- Save states
- Standalone native installers
- PyPI publishing

## Running from standalone executable

Download the executable for your platform from the GitHub Release, then run:

```bash
# Linux
chmod +x ./nes-py-linux-x86_64
./nes-py-linux-x86_64 --help
./nes-py-linux-x86_64 path/to/game.nes
```

```powershell
# Windows PowerShell
.\nes-py-windows-x86_64.exe --help
.\nes-py-windows-x86_64.exe path\to\game.nes
```

The standalone executable bundles Python and project dependencies. It does not
require cloning the repository or creating a virtual environment.

## Installation from Python package artifact

Download the wheel from the GitHub Release, then install it with:

```bash
python -m pip install nes_py-0.0.1-py3-none-any.whl
```

Then verify:

```bash
python -m nes_py --help
python -m nes_py --version
```

## Status

This release is intended for project setup validation and future development.
It is not yet a complete NES emulator.
