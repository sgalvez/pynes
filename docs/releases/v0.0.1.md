# v0.0.1 - Initial emulator, packaging, and release workflow

Release date: 2026-07-01

Tag: `v0.0.1`

Tag commit: `8c942d2`

## Summary

This release establishes the project as an installable Python NES emulator with
a command-line runner, automated validation, release packaging, and standalone
CLI artifacts. It is still an early emulator, but it includes the first useful
CPU, cartridge, bus, PPU, input, debug, and packaging foundations.

## Added

- Python package scaffold under `src/nes_py`.
- CLI entry points through both `nes-py` and `python -m nes_py`.
- iNES ROM loading with NROM / Mapper 0 support.
- Official-opcode MOS 6502 CPU core.
- NES CPU memory bus with RAM mirroring, cartridge access, PPU registers, APU
  register placeholders, controller reads, and expansion register storage.
- PPU foundation with VRAM, palette memory, scroll/address latches, vblank/NMI
  signaling, and background framebuffer rendering.
- Desktop pygame runner with keyboard controls, pause, reset, quit, ROM loading,
  and optional CPU trace output.
- Headless smoke-test mode for ROM validation without opening a desktop window.
- Debug trace formatting and best-effort instruction disassembly.
- Direct script execution support for `src/nes_py/nes.py`.
- Graceful errors for missing pygame and unsupported cartridges.
- Mapper 1 / MMC1 cartridge bank switching.
- Test coverage for CLI behavior, cartridge loading, CPU opcodes, bus behavior,
  PPU behavior, debug helpers, direct script execution, and version metadata.
- Ruff linting and pytest configuration.
- CI on Windows, Linux, and macOS.
- Continuous build artifacts with build metadata, generated changelog, and
  generated release notes.
- Performance profiling docs and a lightweight profiling script.
- Tag-based release workflow that builds source distribution and wheel
  artifacts.
- Standalone Windows and Linux executables using PyInstaller.
- Manual release workflow dispatch support.

## Changed

- Top-level imports were made lazier so simple CLI startup paths do less work.
- Build metadata generation stamps development versions for continuous builds.
- Release docs are used as the body text for GitHub Releases.

## Fixed

- Missing pygame now reports an installation hint instead of an import traceback.
- Unsupported or mismatched ROM mappers now fail with user-facing cartridge
  errors.
- Direct execution of `src/nes_py/nes.py` now works from a checkout.
- Initial release lint failures were corrected before tagging.

## Validation

The release workflow validates the project with:

- `python -m pytest`
- `python -m ruff check .`
- `python -m nes_py --help`
- `python -m nes_py --version`
- `python -m build`
- standalone executable build and `--help` / `--version` smoke checks

## Known Limitations

- Mapper support is incomplete.
- PPU behavior is not cycle-accurate.
- Sprite rendering is not included in this release.
- APU/audio playback is not included in this release.
- Save states and native installers are not included.
