# v0.0.5 - Cache scrolled backgrounds for faster rendering

Release date: 2026-07-01

Tag: `v0.0.5`

Tag commit: `340b211`

Compare: `v0.0.4...v0.0.5`

## Summary

This release recovers performance lost during scroll-aware rendering by caching
the rendered background and reusing it until the background state changes. It
also removes the previous audio smoothing experiment because it degraded sound
quality.

## Added

- Cached background framebuffer storage.
- Cached packed RGB background bytes.
- Tests proving cached background reuse until scroll changes.

## Changed

- `render_frame()` now copies a cached background when possible before drawing
  sprites.
- Full background redraws happen only after relevant PPU state changes.
- Audio output returns to the unsmoothed mixer path.

## Fixed

- Scrolled-background rendering no longer redraws every frame when the
  background state is unchanged.
- Audio is less muffled than in `v0.0.4`.

## Validation

- `python -m pytest`
- `python -m ruff check .`
- release workflow package and standalone artifact checks

## Known Limitations

- Games that write scroll or VRAM every frame can still force full background
  redraws.
- PPU rendering remains an approximation rather than cycle-accurate rendering.
