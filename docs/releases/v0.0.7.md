# v0.0.7 - Fix scaled pygame rendering crash

Release date: 2026-07-01

Tag: `v0.0.7`

Tag commit: `7d0c48c`

Compare: `v0.0.6...v0.0.7`

## Summary

This release fixes a Windows standalone crash introduced by the scaled-surface
optimization in `v0.0.6`. Pygame requires destination surfaces passed to
`transform.scale()` to match the source format, so the runner now uses the safe
two-argument scaling path again.

## Added

- Fake-pygame regression test covering scaled desktop rendering without a real
  pygame dependency.

## Changed

- Scaled rendering now calls `pygame.transform.scale(surface, size)` without an
  incompatible destination surface.

## Fixed

- Windows standalone execution no longer fails with:

```text
ValueError: Source and destination surfaces need the same format.
```

## Validation

- `python -m pytest`
- `python -m ruff check .`
- standalone executable build
- standalone executable `--help` and `--version` smoke checks

## Known Limitations

- The performance improvements from `v0.0.6` remain, except for the reverted
  scaled-surface reuse optimization.
- A real ROM smoke test is still manual because ROM files are not committed to
  the repository.
