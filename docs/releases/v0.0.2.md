# v0.0.2 - Improve frame pacing and add pulse audio

Release date: 2026-07-01

Tag: `v0.0.2`

Tag commit: `3bb9009`

Compare: `v0.0.1...v0.0.2`

## Summary

This release makes desktop execution closer to normal emulator timing by running
until an emulated video frame completes, pacing the pygame window at 60 FPS, and
starting the APU audio path with pulse-channel PCM output.

## Added

- `NES.run_frame()` to execute CPU instructions until the PPU advances one
  frame.
- Frame-paced desktop execution for the default pygame runner.
- Initial APU pulse-channel register handling and PCM sample generation.
- Desktop audio playback through pygame mixer.
- Tests for frame execution and generated audio samples.

## Changed

- The desktop runner now defaults to one emulated video frame per draw instead
  of a fixed instruction count.
- `--instructions-per-frame` remains available as an override for debugging and
  smoke validation.

## Fixed

- Gameplay no longer depends solely on a manually chosen instruction batch size.
- The emulator starts producing audible output for ROMs that use pulse channels.

## Validation

- `python -m pytest`
- `python -m ruff check .`
- release workflow package and standalone artifact checks

## Known Limitations

- Audio is approximate and only covers the first pulse-channel foundation.
- PPU and CPU timing are still not cycle-accurate.
