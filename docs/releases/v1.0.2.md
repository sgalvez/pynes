# v1.0.2 - Reduce metallic audio aliasing

Release date: 2026-07-02

Tag: `v1.0.2`

Compare: `v1.0.1...v1.0.2`

## Summary

This release is a final focused audio pass after `v1.0.1`. Pygame/SDL remains
the desktop playback library because it is already the correct lightweight
choice for the packaged Windows and Linux executables. The metallic sound was
coming mainly from the emulator-side waveform approximation: hard pulse edges,
stepped triangle output, and too-prominent noise entering the mixer.

`v1.0.2` smooths those generated waveforms before they become signed 16-bit PCM
and locks SDL to the same mono format the APU emits.

## Changelog Since v1.0.1

### Audio

- Added a compact PolyBLEP correction for pulse-channel transitions to reduce
  aliasing from hard square-wave edges.
- Replaced the 32-step triangle output with a continuous triangle approximation.
- Added per-channel noise smoothing before the noise channel enters the
  pulse/TND mixer.
- Reduced the noise-channel mix level again so effects are less dominant over
  melody lines.
- Kept the existing DC blocker, low-pass filter, and soft clipper from
  `v1.0.1`.
- Locked pygame/SDL mixer initialization to the signed 16-bit mono PCM format
  emitted by `APU.generate_samples_into()`.
- Increased the SDL mixer buffer to reduce crackle when frame rendering briefly
  overruns.

### Code Documentation

- Documented the PolyBLEP edge correction helper.
- Documented why the desktop runner pins SDL mixer settings.
- Updated the APU class docstring to describe the actual PCM output role.

### Packaging And Releases

- Bumped package version to `1.0.2`.
- Updated README release examples, artifact names, and release tag commands for
  `v1.0.2`.
- Added this permanent release note and linked it from the release index.

### Tests

- Added tests for the PolyBLEP transition correction.
- Added tests that pulse, triangle, and noise channel outputs use bounded
  smoothed float shapes.
- Updated version metadata and release documentation coverage for `v1.0.2`.

## Known Limitations

- CPU, PPU, and APU behavior are still not cycle-accurate.
- APU envelopes, sweeps, length counters, DMC, and exact frame-counter behavior
  are not complete.
- The smoother oscillator approximation should reduce metallic aliasing, but it
  is still not a full NES audio emulation core.
- Real ROM smoke tests remain manual because ROMs are not committed to the
  repository.
