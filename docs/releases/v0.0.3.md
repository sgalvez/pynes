# v0.0.3 - Render sprites and improve APU mixing

Release date: 2026-07-01

Tag: `v0.0.3`

Tag commit: `467a8a7`

Compare: `v0.0.2...v0.0.3`

## Summary

This release makes character and object layers visible by adding OAM sprite
rendering and DMA support. It also broadens audio output by adding triangle and
noise channel approximations to the APU mixer.

## Added

- 8x8 sprite rendering from PPU OAM.
- Sprite palette lookup, transparency, horizontal/vertical flips, and basic
  priority handling.
- `$4014` OAM DMA support on the NES bus.
- Triangle-channel audio approximation.
- Noise-channel audio approximation.
- NES-style pulse/TND mixing formulas for more balanced output.
- Tests for sprite rendering, OAM DMA, and triangle/noise audio output.

## Changed

- The PPU frame render path now composites sprites over the background.
- APU output mixes pulse, triangle, and noise channels instead of pulse only.

## Fixed

- ROMs that drew the player or enemies as sprites no longer show only the
  background layer.
- Audio has more recognizable channel coverage for games using triangle or noise.

## Validation

- `python -m pytest`
- `python -m ruff check .`
- release workflow package and standalone artifact checks

## Known Limitations

- Sprite rendering is still simplified and not cycle-accurate.
- 8x16 sprite mode and several PPU edge cases remain incomplete.
- APU envelopes, sweeps, length counters, and DMC are not complete.
