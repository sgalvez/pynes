# v1.0.1 - Smooth overloaded frames and soften audio peaks

Release date: 2026-07-02

Tag: `v1.0.1`

Compare: `v1.0.0...v1.0.1`

## Summary

This release continues the playability work from `v1.0.0`. It focuses on the
two remaining rough edges reported during ROM smoke testing: frame spikes when
scenes become busy or swap screens, and the harsh metallic edge still audible in
the approximate APU output.

The emulator is still not cycle-accurate, but this version does less repeated
background decoding during redraw-heavy frames, skips more sprite work before
touching pixel loops, and rounds off sharper audio peaks before samples reach
pygame.

## Changelog Since v1.0.0

### Runtime And Video Performance

- Kept decoded background tile rows in a bounded PPU cache across background
  redraws instead of rebuilding the same rows on every dirty-frame pass.
- Invalidated the persistent tile-row cache when CHR or palette writes can
  change the rendered pixels.
- Bounded the tile-row cache so ROMs with aggressive CHR animation cannot grow
  memory without limit.
- Changed sprite scanline selection to store visible row masks per sprite. This
  lets crowded scenes skip sprites excluded by the eight-sprites-per-scanline
  rule before entering the heavier row and pixel rendering loops.
- Added focused tests proving background tile rows are reused across redraws.

### Audio

- Lowered the post-mix low-pass filter alpha so harsh edges are smoothed a bit
  more gradually.
- Reduced the final output gain to leave more headroom before sample clipping.
- Reduced the noise-channel mix level so noise effects do not dominate pulse and
  triangle melodies.
- Added soft clipping before integer sample conversion to round off sharp peaks
  instead of hard-clamping them.
- Added tests around the soft clipper and existing audio reset behavior.

### Code Documentation

- Documented why the APU output shaping constants exist.
- Documented the persistent PPU tile-row cache and the reason it is bounded.
- Kept sprite selection documented around the NES eight-sprite scanline limit.

### Packaging And Releases

- Bumped package version to `1.0.1`.
- Updated README release examples, artifact names, and release tag commands for
  `v1.0.1`.
- Added this permanent release note and linked it from the release index.

## Known Limitations

- CPU, PPU, and APU behavior are still not cycle-accurate.
- APU envelopes, sweeps, length counters, and DMC are not complete.
- PPU scrolling and screen transitions remain simplified and can still stress
  redraw-heavy games.
- Mapper support is still limited.
- Real ROM smoke tests remain manual because ROMs are not committed to the
  repository.
